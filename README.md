# Documentacion
## Resumen Ejecutivo
### Descripción del Programa
“Tienda de Abarrotes Lupita” es una solución de punto de venta (POS) diseñada para tiendas de abarrotes y otros negocios comerciales. La aplicación facilita la gestión de ventas e inventarios, ofreciendo una interfaz web intuitiva y eficiente. El sistema está diseñado para pequeños comercios que buscan modernizar y optimizar sus operaciones diarias, reemplazando métodos manuales como el uso de hojas de cálculo.
### Problema Identificado
Las pequeñas tiendas de abarrotes a menudo carecen de un sistema automatizado para gestionar las ventas y controlar el inventario. Esto puede resultar en una gestión ineficiente, errores humanos y una falta de visibilidad en tiempo real sobre el stock de productos. El uso de sistemas manuales, como hojas de Excel, bitácoras físicas; puede ser engorroso y propenso a errores en todas las áreas de esta.

## Solución
La solución propuesta para las necesidades de gestión de la “Tienda de Abarrotes Lupita” es una aplicación web de punto de venta (POS) que ofrece las siguientes funcionalidades clave:
### 01.- Gestión de Ventas:
- Interfaz de Caja Rápida: Permite a los cajeros registrar transacciones de venta de manera - eficiente, con la capacidad de añadir productos al carrito, aplicar descuentos y procesar pagos. La interfaz está diseñada para minimizar el tiempo de atención al cliente y facilitar un flujo de trabajo rápido.
- Historial de Ventas: Mantiene un registro detallado de todas las transacciones realizadas, incluyendo fecha, hora, productos vendidos, método de pago y cajero responsable. Esto ayuda en el seguimiento de ventas y auditorías.
### 02.- Gestión de Inventario:
- Actualización Automática de Inventario: Cada vez que se realiza una venta, el inventario se actualiza automáticamente para reflejar la cantidad de productos restantes. Esto evita la sobreventa y ayuda a mantener un control preciso del stock.
- Alerta de Stock Bajo: El sistema genera alertas cuando el nivel de inventario de un producto alcanza un umbral predefinido, lo que permite a los gerentes de la tienda realizar pedidos a tiempo y evitar faltantes.
- Gestión de Productos: Incluye funcionalidades para añadir, editar y eliminar productos del catálogo, así como para gestionar sus precios, descripciones y categorías.
### 03.- Gestión de Usuarios:
- Control de Acceso: Implementa un sistema de autenticación y autorización que asegura que solo el personal autorizado pueda acceder a determinadas funcionalidades, como la gestión de inventario o la visualización de reportes financieros.
- Roles y Permisos: Diferentes roles (por ejemplo, administrador, cajero) con permisos específicos pueden ser asignados a los usuarios para garantizar un control adecuado y la seguridad del sistema.
### 04.- Reportes y Análisis:
- Reportes de Ventas: Genera reportes diarios, semanales y mensuales que muestran las ventas totales, productos más vendidos, horas pico de ventas, y otros datos relevantes. Estos reportes pueden exportarse en formato CSV para su análisis.
- Análisis de Inventario: Proporciona insights sobre el movimiento de productos, permitiendo a los gerentes identificar tendencias de venta y ajustar sus estrategias de inventario.
### 05.- Integraciones Futuras (Prospección):
- Escáner de Códigos de Barras: Se planea integrar la funcionalidad para escanear códigos de barras, lo que agilizará aún más el proceso de venta y gestión de inventario.
- Módulo de Gestión de Proveedores: En el futuro, se añadirá un módulo para gestionar pedidos y relaciones con proveedores, facilitando el reabastecimiento de productos.
### 06.- Escalabilidad y Despliegue (Prospección):
- Despliegue en la Nube: La aplicación puede ser desplegada en servidores locales o en la nube, utilizando plataformas como Heroku. Esto ofrece flexibilidad en términos de escalabilidad y accesibilidad.
- Modularidad y Extensibilidad: La arquitectura del sistema está diseñada de manera modular, permitiendo la fácil adición de nuevas funcionalidades sin afectar la operación existente.

## Arquitectura
El Proyecto “Tienda de Abarrotes Lupita” está diseñado con una arquitectura de tres capas, que incluye la capa de presentación, la capa de lógica de negocio y la capa de datos. Esta arquitectura modular permite una separación clara de responsabilidades y facilita el mantenimiento y la escalabilidad - evolución del sistema.
### 01. Capa de Presentación
Tecnologías: JavaServer Pages (JSP), HTML, CSS, JavaScript
- Interfaz de Usuario: La interfaz de usuario se implementa utilizando JSP, combinada con HTML, CSS y JavaScript para crear páginas web dinámicas y responsivas. Estas páginas permiten a los usuarios interactuar con el sistema de manera intuitiva y eficiente. La capa de presentación se comunica con la capa de lógica de negocio para solicitar datos y realizar acciones, como registrar ventas o actualizar el inventario.
- Controladores de JSP: Los controladores de JSP manejan la navegación y las interacciones del usuario, procesando las solicitudes HTTP y respondiendo con las vistas adecuadas. Esto incluye la gestión de formularios de entrada, la validación de datos y la gestión de sesiones de usuario.
### 02. Capa de Lógica de Negocio
Tecnologías: Java Servlets, JavaBeans, JDBC
- Servidores de Aplicaciones: La lógica de negocio se ejecuta en un servidor Tomcat, donde se despliegan los servlets Java que manejan las operaciones principales del sistema. Los servlets actúan como intermediarios entre la capa de presentación y la capa de datos, gestionando la lógica empresarial.
- JavaBeans: Los componentes JavaBeans encapsulan la lógica de negocio y los datos del sistema. Estos componentes se utilizan para representar entidades del sistema como productos, usuarios, y transacciones. Cada JavaBean incluye métodos para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) y otras funciones específicas del negocio.
- Gestión de Transacciones: El sistema garantiza la integridad de las transacciones mediante la implementación de mecanismos de control de transacciones. Esto asegura que todas las operaciones de compra, actualización de inventario y otras acciones críticas sean completadas correctamente o revertidas en caso de error.
### 03. Capa de Datos
Tecnologías: MySQL, JDBC
- Base de Datos: La base de datos MySQL almacena toda la información relacionada con los productos, usuarios, ventas, y otras entidades del sistema. Se utiliza una estructura de tablas relacionales normalizadas para asegurar la eficiencia y consistencia de los datos.
- Conexión a la Base de Datos: La conexión entre la aplicación y la base de datos se maneja a través de JDBC (Java Database Connectivity). Esto permite que los servlets y JavaBeans interactúen con la base de datos de manera segura y eficiente, ejecutando consultas SQL para recuperar, insertar, actualizar o eliminar datos.
### 04. Integración y Seguridad
- Autenticación y Autorización: El sistema implementa mecanismos de autenticación para validar las credenciales de los usuarios y controlar el acceso a diferentes partes de la aplicación según el rol del usuario (administrador, cajero, etc.). Esto se maneja mediante sesiones de usuario y filtros de seguridad en la configuración del servidor.
- Encriptación de Datos: Se recomienda el uso de SSL/TLS para encriptar las comunicaciones entre el cliente y el servidor, especialmente para proteger la transmisión de información sensible como contraseñas y datos personales.
### 05. Despliegue y Mantenimiento
- Despliegue en Tomcat: La aplicación se empaqueta como un archivo WAR (Web Application Archive) y se despliega en un servidor Apache Tomcat. Esto facilita el despliegue en diferentes entornos, desde servidores locales hasta servicios en la nube como Heroku.
- Escalabilidad y Extensibilidad: La arquitectura modular del sistema permite una fácil escalabilidad horizontal, añadiendo más instancias del servidor de aplicaciones según sea necesario. Además, la extensibilidad se facilita mediante la adición de nuevos módulos o la modificación de los existentes sin afectar las funcionalidades básicas del sistema.

<a name="Requerimientos"></a>
# Requerimientos
## Servidores
### 01.- Servidor de Aplicación:
* Apache Tomcat: Se requiere Tomcat 8 o superior para desplegar la aplicación web. Tomcat es un contenedor de servlets y JSP, responsable de ejecutar las aplicaciones Java EE y proporcionar servicios como manejo de sesiones, recursos de red y seguridad.
### Configuracion Recomendada: 
- RAM: Al menos 2 GB de RAM para un rendimiento adecuado en entornos de producción pequeños.
- CPU: Procesador de al menos 2 núcleos.
- Almacenamiento: Espacio de disco suficiente para logs y datos temporales, aproximadamente 10 GB.
### 02.- Servidor Web (Opcional): 
- Si se requiere una configuración más robusta o un entorno de producción, se puede utilizar un servidor web como Apache HTTP Server o Nginx como proxy inverso. Estos servidores pueden manejar la carga de las solicitudes HTTP y HTTPS, balanceo de carga, y otros servicios de red.

### 03.- Servidor de Aplicación:
* MySQL: Se utiliza MySQL como sistema de gestión de bases de datos relacional (RDBMS). MySQL versión 5.7 o superior es recomendada.
### Configuracion Recomendada: 
- RAM: Al menos 1 GB de RAM para bases de datos pequeñas.
- CPU: Procesador de al menos 2 núcleos.
- Almacenamiento: Dependiendo del volumen de datos, se recomienda comenzar con 20 GB y ajustar según el crecimiento.





























<a name="Instalacion"></a>
### ¿Cómo instalar el ambiente de desarrollo?
Se debe agregar el proyecto (WAR) en el App Manager de tomcat y desplegarlo y crear la base de datos con los archivos sql.

## Paquetes Adicionales.
### 01. Conector JDBC:
* MySQL Connector/J: Se requiere el conector JDBC para MySQL (versión 5.1.33 o superior) para permitir que la aplicación Java se comunique con la base de datos MySQL. Este conector debe estar disponible en el classpath del servidor de aplicaciones.

### 02. Librerías Java:
* JavaBeans Activation Framework (JAF): Necesario si se utiliza JavaMail o para manejar tipos MIME.
* JavaMail API: Si se requiere enviar correos electrónicos desde la aplicación, como confirmaciones de pedidos o notificaciones.

### 03. Herramientas de Desarrollo:
* IDE de Java: Como NetBeans, Eclipse o IntelliJ IDEA para el desarrollo y mantenimiento del código.
* Apache Ant o Apache Maven: Para la automatización de la compilación y gestión de dependencias.

### 04. Control de Versiones:
*Git: Para el control de versiones y la colaboración en el desarrollo del proyecto. Se recomienda GitHub como plataforma de alojamiento de repositorios.

## Version de Java, Etc.
### 01. Java:
- Java Development Kit (JDK): Se requiere JDK 8 (Java 1.8) para el desarrollo y ejecución de la aplicación. Es crucial asegurar que tanto el entorno de desarrollo como el entorno de producción utilicen la misma versión para evitar problemas de compatibilidad.

### 02. Sistemas Operativos:
- Compatibilidad: La aplicación es compatible con sistemas operativos basados en UNIX/Linux y Windows. Se recomienda un entorno Linux para servidores de producción debido a su estabilidad y eficiencia en el manejo de recursos.

### 03. Navegadores Compatibles:
- Actualización Constante: La interfaz web del sistema es accesible desde los navegadores modernos como Google Chrome, Mozilla Firefox, Microsoft Edge, y Safari. Se recomienda mantener actualizados los navegadores para garantizar la compatibilidad y la seguridad.

### Consideraciones Adicionales:
- Seguridad: Es recomendable implementar HTTPS para asegurar la comunicación entre los clientes y el servidor. Para esto, se pueden utilizar certificados SSL/TLS.
- Backups y Recuperación: Se debe establecer un plan de backups regulares para la base de datos y los archivos críticos del sistema. Además, es importante tener procedimientos claros para la recuperación ante desastres.

# Instalacion
## Configuración del Entorno de Desarrollo
### 01a.- Instalar JDK 8:
- Descarga e instala JDK 8 desde Oracle o utiliza OpenJDK.
- Configura la variable de entorno JAVA_HOME apuntando al directorio de instalación de JDK.
### 01b. Instalar Apache Tomcat 8:
- Descarga Apache Tomcat 8 desde Apache Tomcat.
- Extrae el archivo descargado y configura la variable de entorno CATALINA_HOME apuntando al directorio de Tomcat.
### 01c. Instalar MySQL:
- Descarga e instala MySQL desde MySQL.
- Configura MySQL y crea una base de datos para el proyecto.
### 01d. Instalar IDE:
- Instala un IDE como IntelliJ IDEA, Eclipse o NetBeans. Configura el IDE para usar JDK 8 y Apache Tomcat.
### 01e. Clonar el Repositorio:
- Clona el repositorio del proyecto desde GitHub.

## Paso 2: Configuración de Dependencias:
### 02a.- Configurar MySQL Connector/J:
- Descarga el conector JDBC para MySQL desde MySQL Connector/J.
- Copia el archivo mysql-connector-java-5.x.x-bin.jar en el directorio lib de Tomcat ($CATALINA_HOME/lib).
### 02b.- Configurar el Proyecto en el IDE:
- Importa el proyecto clonado en tu IDE y configura el servidor Tomcat en las configuraciones de ejecución del IDE.
- Asegúrate de que las dependencias y configuraciones del proyecto estén correctamente configuradas.

## Paso 3: Configuración de la Base de Datos:
### 03a.- Configurar el archivo de propiedades:
- En el directorio src/main/resources, crea o edita el archivo database.properties con las credenciales de la base de datos.
### 03b.- Inicializar la Base de Datos:
- Ejecuta los scripts SQL proporcionados en el directorio sql del proyecto para crear las tablas necesarias y poblar la base de datos con datos iniciales.

## ¿Cómo ejecutar pruebas manualmente?
### Paso 1: Iniciar Apache Tomcat
### 01a.- Instalar JDK 8:
- Inicia el servidor Tomcat desde el IDE o manualmente ejecutando el script de inicio (startup.bat para Windows, startup.sh para Linux/Mac) ubicado en el directorio bin de Tomcat.

## Paso 2: Acceder a la Aplicación
### 02a.- Abrir el Navegador:
- Abre un navegador web y navega a http://localhost:8080/Proyecto-Abarrotes-.
### 02b.- Pruebas de Funcionalidad:
- Login: Inicia sesión con las credenciales de administrador para acceder al panel de control.
- Gestión de Productos: Agrega, edita y elimina productos en el inventario. Verifica que las acciones se reflejen correctamente en la base de datos.
- Registro de Ventas: Simula una venta, añadiendo productos al carrito y procesando el pago. Verifica que el inventario se actualice adecuadamente.
- Generación de Reportes: Accede a las secciones de reportes para visualizar y exportar datos de ventas y stock.

### 01b.- Desplegar en Tomcat.

### 01c.- Configuración del Entorno de Producción.
- Configura las propiedades de la base de datos y otras configuraciones específicas de producción en los archivos de propiedades o mediante variables de entorno.

### 01a.- Pre-requisitos:

### 01b.- Configurar Heroku CLI:
- Inicia sesión en Heroku.
### 01c.- Crear una Aplicación en Heroku:
- Navega al directorio del proyecto y crea una nueva aplicación.
### 01d.- Configurar MySQL en Heroku:

- Añade los archivos necesarios para el despliegue en Heroku (Procfile, system.properties)
